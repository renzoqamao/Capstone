Incovenientes con la Implementación

Sincronización con los distintos proyectos de GNS3

Para realizar el proyecto Capstone tuvimos que dividirmos por sedes, con el fin de avanzar con el entregable, pero tuvimos dificultad al unir las distintas configuraciones de las topologias.

Solucion:

Se dicidió que un integrante de nuestro grupo ejecutara, las distintas topologias que realizó cada integrante en distintas gns3 en su computadora y comprobara las conexiones a través de nodo cloud GNS3 usando tuberia UDP.

La cantidad de IP que se tiene que asignar a los distintos host

Solución:

Se aplicó el concepto de subneting para designar el rango de IP para cada sede (Trujillo, Central, Atención al Cliente) y luego se realizó subneteo VLSM para cada sede.

Inconveniencia con el Router

El routers (C3725) que establecimos al comienzo en cada sede tenían pocos interfaces que venia por defectos .

Solución:

Se buscó otro Router que tenía mas interfaces para la conexiones que necesitabamos en cada sede, el Router que elegimos fue C7200

Topologia

Para la realización de la topologia se analizó primero la topologia lógica, pero no se tuvo encuenta la topologia física, entonces al concluir la topologia lógica pasamos a la topologia física donde se observó algunos inconveniencias.

Solución:

Se decidió realizar la topología física y partiendo de ella se realizó algunas modificaciones en la topología lógica
